# 📂 NYPDOnline.org Documentation Archive

A **daily-updated archive of publicly available documents from NYPDOnline.org**.

This repository automatically collects and preserves files published on the website in order to maintain a **historical record of documents over time**. The archive provides a reliable backup of materials that may otherwise be modified, relocated, or removed from the source website.

By using Git version control, the repository keeps track of **file changes, updates, and historical versions**, helping ensure that important public records remain accessible.

---

# 🎯 Project Goals

The main goals of this repository are to:

📦 **Archive Public Documents**
Maintain copies of publicly accessible files published on NYPDOnline.org.

🕒 **Preserve Historical Records**
Keep historical versions of documents so changes can be tracked over time.

🔄 **Automate Daily Backups**
Ensure files are archived regularly through automated scripts.

🛟 **Provide Backup & Recovery**
Create a reliable fallback if documents are removed or modified on the original site.

🔍 **Support Transparency & Research**
Help journalists, researchers, legal professionals, and community organizations access historical records.

🌐 **Provide Open Access to Archived Files**
Make archived documents easily available for anyone interested in reviewing them.

# 🗂 What This Repository Contains

This archive stores **publicly accessible documents from NYPDOnline.org**, including but not limited to:

- PDF documents
- Forms
- Reports
- Public information files
- Policy documents
- Downloadable resources
- Other files made available through the website

Files are organized in folders based on their **source location or download path**, helping maintain structure similar to the original website.

---

# ⚙️ How The Archive Works

This repository uses automated tools and scripts to collect and store documents.

The general process works as follows:

1️⃣ The website is scanned for publicly available documents
2️⃣ Files are downloaded locally
3️⃣ New or changed files are identified
4️⃣ The repository is updated with the latest versions
5️⃣ Git records all changes for historical tracking

This system ensures that the repository maintains a **timeline of document updates** over time.

---

# 📅 Update Schedule

⏱ **Automated Daily Updates**

The archive is refreshed **once per day** using automated processes.

During each update:

- ➕ Newly discovered files are downloaded
- 🔄 Updated files are replaced and recorded
- 🗃 Historical versions remain accessible through Git history

This allows the repository to function as a **living archive** of the website.

---

# 🕰 Version History

One of the key benefits of using Git is the ability to track document changes over time.

Git allows users to:

- View previous versions of files
- Compare document changes
- Identify when updates occurred
- Restore older versions if necessary

This creates a **transparent historical timeline of file updates**.
